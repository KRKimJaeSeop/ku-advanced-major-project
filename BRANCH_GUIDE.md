# 브랜치 사용 가이드

## 브랜치 구조

| 브랜치 | 용도 |
|---|---|
| `main` | 최종 제출용. 직접 작업 금지 |
| `develop` | 개인 작업물을 합치는 통합 브랜치 |
| `jaesub` | 김재섭 개인 작업 브랜치 |
| `daewon` | 김대원 개인 작업 브랜치 |
| `arim` | 윤아림 개인 작업 브랜치 |
| `jimin` | 이지민 개인 작업 브랜치 |

---

## 작업 흐름

```
개인 브랜치에서 작업 → develop으로 PR → 확인 후 병합
```

1. **본인 브랜치**에서 작업한다
2. 작업이 완료되면 `develop` 브랜치로 **Pull Request(PR)** 를 연다
3. 팀원 한 명 이상이 확인한 뒤 병합한다
4. 주기적으로 `develop`의 최신 내용을 본인 브랜치로 **당겨온다(Pull)**

---

## 주의사항

- `main`과 `develop`에 **직접 push 금지**
- 반드시 PR을 통해서만 병합한다
- 작업 시작 전에 항상 본인 브랜치가 최신 상태인지 확인한다

---

## 소스트리 기준 작업 순서

1. 왼쪽 사이드바에서 본인 브랜치 더블클릭 (체크아웃)
2. 작업 후 **Commit**
3. 상단 **Push** 버튼으로 원격 저장소에 올리기
4. GitHub에서 `develop`으로 **Pull Request** 생성
